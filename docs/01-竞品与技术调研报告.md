### 1.3 亮点与短板

**ShareX** — 功能之王，Windows独占
- 亮点: 15+截图模式、80+上传目标、工作流自动化、滚动截图、OCR
- 短板: 仅Windows、UI复杂、学习曲线陡、无贴图

**CleanShot X** — macOS天花板
- 亮点: 滚动截屏、截图置顶、自毁截图、系统音频录制、标注专业
- 短板: 仅macOS、不开源、$29买断（截屏$19）

**Snipaste** — 贴图革命者
- 亮点: F1截图/F3贴图极简流、贴图支持缩放/旋转/透明度、取色器专业级
- 短板: 无滚动截图、无录屏、Mac收费、标注基础

**Greenshot** — 经典可靠的自由软件
- 亮点: 完全开源、跨平台、Office集成、企业部署友好
- 短板: 无录屏/GIF、标注基础、UI老旧

**Flameshot** — Linux标配
- 亮点: CLI自动化、插件系统、标注美观、完全开源
- 短板: 跨平台体验不一致、无录屏、无滚动截图

### 关键发现

- **截图+录屏一体化是大趋势**: CleanShot X、Snagit 均同时支持
- **GPU硬件编码是刚需**: NVENC/AMF/VideoToolbox/QSV 决定录制流畅度
- **轻量级录屏存在空白**: OBS太重、Loom依赖云端、Kap仅Mac
- **跨平台录屏工具极度稀缺**: OBS几乎是唯一选择，但功能过于复杂

## 三、技术栈全景分析

### 3.1 各平台原生屏幕捕获 API

#### Windows
| API | 性能 | 适用场景 |
|-----|------|---------|
| **DXGI Desktop Duplication** | ★★★★★ | 屏幕录屏，GPU直取帧缓冲 |
| **Windows Graphics Capture (WGC)** | ★★★★ | Win10+，支持窗口隔离捕获 |
| **BitBlt (GDI)** | ★★★ | 基础截图，传统兼容 |

#### macOS
| API | 性能 | 适用场景 |
|-----|------|---------|
| **ScreenCaptureKit** | ★★★★★ | macOS 13+，官方推荐，支持窗口选择+音频 |
| **Core Graphics (CGDisplay)** | ★★★★ | 传统截图，兼容性好 |
| **AVFoundation + VideoToolbox** | ★★★★★ | 硬件编码(H.264/H.265) |

#### Linux
| API | 性能 | 适用场景 |
|-----|------|---------|
| **PipeWire + XDG Desktop Portal** | ★★★★★ | Wayland 现代标配 |
| **X11 (XCB/Xlib)** | ★★★ | X11 传统兼容 |
| **DRM/KMS** | ★★★★★ | 直接帧缓冲(需root) |

### 3.2 Rust 生态关键库

#### 屏幕捕获（推荐: **xcap**）
| Crate | 平台覆盖 | 能力 | 成熟度 |
|-------|---------|------|--------|
| **xcap** | Win/Mac/Linux | Monitor/Window枚举 + 截图 + Video Recording | ⭐⭐⭐⭐ |
| captrs | Win/Linux | 基础截图(DXGI+X11) | ⭐⭐⭐ |
| screencapturekit-rs | macOS | ScreenCaptureKit Rust绑定 | ⭐⭐⭐ |
| dxgi-capture-rs | Windows | DXGI纯绑定 | ⭐⭐⭐ |

#### GUI 框架（推荐: **egui**）
| 框架 | 渲染后端 | 适合场景 |
|------|---------|---------|
| **egui** | GL/Metal/DX | 即时模式，轻量，完美适配覆盖层 |
| iced | wgpu | Elm架构，组件化 |
| tauri | WebView | 前端+系统API，生态好 |
| Slint | 原生 | 声明式，嵌入式友好 |

#### AI增强 / OCR引擎
| Crate | 用途 | 优势 |
|-------|------|------|
| **ort** + PaddleOCR ONNX | OCR主力（中英文） | 精度>95%、纯本地、模型<10MB |
| **tesseract-plumbing** + leptonica-sys | OCR fallback（多语言） | 100+语言、排版分析强 |
| **rqrr** | 二维码/条形码 | 纯Rust、零依赖 |

##### OCR 引擎对比

| 引擎 | 方式 | 中文精度 | 部署复杂度 | 模型体积 | Rust绑定 |
|------|------|---------|-----------|---------|---------|
| **PaddleOCR (ONNX)** | ONNX Runtime | >95% | 中（需ort+模型下载） | ~10MB | `ort` crate |
| **RapidOCR** | ONNX/OpenVINO | >94% | 中（无Rust绑定，需subprocess） | ~10MB | ❌（需FFI） |
| **Tesseract 5** | LSTM Neural | ~85% | 低（系统自带或brew/apt） | 50-200MB(语言包) | `tesseract-plumbing` |
| **Google Vision API** | 云API | >97% | 低（但需联网） | 0 | HTTP client |

**选型结论**: PaddleOCR ONNX 通过 `ort` crate 直接加载推理，无需 Python 环境，Rust 原生调用，中文精度最高。Tesseract 作为多语言 fallback。

#### 系统集成
| Crate | 用途 |
|-------|------|
| **tray-icon** | 系统托盘 |
| **global-hotkey** | 全局快捷键 |
| **arboard** | 剪贴板 |
| **notify-rust** | 系统通知 |


## 四、市场机会 (APPEALS 分析)

| 维度 | 市场现状 | 刹那机会 |
|------|---------|---------|
| **$ 价格** | 好工具平台锁定，高级功能付费 | **开源免费 + 跨平台** |
| **A 可获得性** | 跨平台仅Greenshot(功能弱) | **Win/Mac/Linux 一致体验** |
| **P 性能** | 无Rust原生工具 | **启动<100ms/内存<50MB/GPU加速** |
| **E 易用性** | ShareX复杂/Snipaste功能少 | **简洁起点 + 强大功能** |
| **A 可靠性** | 各平台体验不一 | Rust内存安全，零崩溃 |
| **L 生命周期** | 商业工具需续费 | 开源 + 社区 = 长期可用 |

## 五、结论

**核心差异化定位**: 第一款 Rust 原生、跨平台(Win/Mac/Linux)、截图+录屏一体化的开源工具。填补"快速截图贴图 + 轻量录屏"的跨平台空白。
