# 刹那 (Chana) — 需求管理宽表

> 维护日期：2026-06-20 | 维护人：PM Agent Team | 对应 PRD v1.0

## 使用说明

本宽表是 `02-PRD-刹那产品需求文档.md` 的执行视图，用于**开发排期、进度跟踪、验收核对**。
- 每条需求有独立 ID（与 PRD 一致）
- **状态流转**: `规划中 → 研发中 → 测试中 → 已交付`
- **Milestone 列**对应研发计划的 Demo / MVP / 1.0 / 1.5 / 2.0
- 建议配合 GitHub Issue/Kanban 使用

## 需求依赖关系

```
MVP 核心管道:
  M03(全屏) ──┐
  M02(窗口) ──┼── M01(区域) ─► M04(标注) ─► M05(剪贴板) + M06(本地保存)
               │
               └── M07(快捷键) + M08(托盘)  ← 全局基础设施
               M09(区域录屏) ─► M10(录屏含音频)  ← 独立管道

1.0 增量:
  M04(标注) ─► O07(标注历史)   ← 标注可编辑需标注引擎完成
  M01 ─► O02(长截图)           ← 长截图基于区域选择
  O01(贴图) 独立模块
  O04(OCR) 独立模块 ─► O05(取色器) 基础可复用
  M09 ─► O06(GIF) + O10(窗口录制)  ← 录屏管道扩展

1.5 增量:
  O04(OCR) ─► A04(截图搜索)    ← 搜索依赖OCR索引
  M04(标注) ─► A05(自动美化)    ← 标注引擎扩展
  A01,A02,A03 独立模块

2.0 增量:
  A06(AI抠图) 独立调研/模块
  M09 ─► A07(录屏编辑)         ← 录屏后处理
  A08(共享库) 需服务端支撑
```
