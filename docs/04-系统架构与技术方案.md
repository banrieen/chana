**OCR 引擎选型策略**:

| 引擎 | 推理框架 | 模型 | 优势 | 劣势 | 定位 |
|------|---------|------|------|------|------|
| **PaddleOCR (ONNX)** | ONNX Runtime via `ort` crate | PP-OCRv5/v6 Det+Rec | 中英文精度最高、ONNX加速4-5x、安装包内嵌离线可用 | 仅中英文（可扩展）、模型合计~10MB | 主力引擎 |
| **Tesseract 5** | libtesseract via `tesseract-plumbing` | LSTM Neural Net | 100+语言支持、成熟稳定、排版分析强；系统自带或内嵌 | 中文精度不如PaddleOCR、语言包体积较大 | 多语言 fallback |

**PaddleOCR ONNX 本地运行原理**:

```
截图 → 预处理(归一化/尺寸对齐)
     → Det模型(DB-Net) → 文本区域坐标
     → 逐个区域裁剪 → Rec模型(CRNN/SVTR) → 文字内容
     → 后处理(CTCDecode/行合并) → OCR结果
```

- 所有推理通过 `ort` crate 调用 ONNX Runtime，无需 Python 环境
- 模型文件（.onnx）首次启动自动从 ModelScope/GitHub 下载到 `~/.chana/models/`
- PP-OCRv6_mobile: 检测2.7MB + 识别5.4MB = 总计约8MB
- 推理速度：CPU 单帧 <200ms（主流笔记本），支持 GPU（CUDA/CoreML）加速

### 2.5 OCR 引擎详细设计

#### 2.5.1 引擎调度策略

```rust
/// OCR 引擎统一入口，自动选择最优引擎
pub struct OcrEngine {
    /// 主力：PaddleOCR ONNX（中文精度 >95%，英文 >98%）
    primary: Option<PaddleOcrEngine>,
    /// Fallback：Tesseract（备选，100+ 语言支持）
    fallback: Option<TesseractEngine>,
    /// 预处理管道
    preprocessor: Preprocessor,
    /// 后处理管道
    postprocessor: Postprocessor,
}

impl OcrEngine {
    pub async fn recognize(&self, image: &ImageBuffer) -> Result<OcrResult> {
        let preprocessed = self.preprocessor.process(image)?;

        // 优先使用 PaddleOCR ONNX（中英文场景）
        if let Some(ref engine) = self.primary {
            if let Ok(result) = engine.recognize(&preprocessed).await {
                return self.postprocessor.refine(result);
            }
        }

        // Fallback 到 Tesseract（多语言或不支持场景）
        if let Some(ref engine) = self.fallback {
            return engine.recognize(&preprocessed);
        }

        Err(OcrError::NoEngineAvailable)
    }
}
```

#### 2.5.2 PaddleOCR ONNX 流水线

```
┌──────────┐    ┌──────────────┐    ┌──────────────┐    ┌─────────┐
│ 输入图像  │───▶│  DB-Net 检测  │───▶│ CRNN/SVTR 识别 │───▶│ 文本结果 │
│ (BGRA)   │    │  (Det ONNX)  │    │  (Rec ONNX)   │    │ (JSON)  │
└──────────┘    └──────────────┘    └──────────────┘    └─────────┘
                      │                    │
                文本区域坐标           逐区域 → Resize(320×48)
                [Rect; N]             归一化 → ONNX 推理 → CTC Decode
```

**模型规格**:

| 模型 | 大小 | 兼容 | 说明 |
|------|------|------|------|
| PP-OCRv6_mobile_det | 2.7 MB | ONNX 1.16+ | 文字检测，基于 DB-Net |
| PP-OCRv6_mobile_rec | 5.4 MB | ONNX 1.16+ | 文字识别，基于 SVTR |

#### 2.5.3 图像预处理管道

```rust
struct Preprocessor;

impl Preprocessor {
    /// 自适应预处理：根据图像特征选择策略
    pub fn process(&self, image: &DynamicImage) -> Result<PreprocessedImage> {
        // 1. 灰度化
        let gray = image.to_luma8();

        // 2. 自适应二值化 (Otsu / Sauvola)
        let binary = match self.detect_noise_level(&gray) {
            NoiseLevel::Low => Self::otsu_threshold(&gray),
            NoiseLevel::High => Self::sauvola_threshold(&gray),
        };

        // 3. 去噪（中值滤波）
        let denoised = Self::median_filter(&binary, 3);

        // 4. 倾斜校正（霍夫变换检测角度）
        let corrected = Self::deskew(&denoised);

        Ok(PreprocessedImage { image: corrected, angle: corrected.angle })
    }
}
```

#### 2.5.4 模型打包与自动更新机制

**核心原则：安装即用，离线无忧；联网自动更新。**

```
安装包内嵌模型 (resources/models/)
     │
     ├──▶ 首次启动（离线）
     │    安装时模型已写入 ~/.chana/models/bundled/
     │    └── 直接加载 → OCR 立即可用 ✓
     │
     └──▶ 后续启动（在线）
          检查远端模型版本 registry
          ├── 已是最新 → 使用 bundled 版本
          └── 有新版本 → 后台下载 → 原子替换 ──▶ 下次启动生效
```

**模型打包策略 — 分层模型目录**:

```
~/.chana/models/ocr/
├── bundled/                    # 安装包内嵌（只读，不回写）
│   ├── ppocrv6_mobile_det.onnx   2.7 MB  sha256:abc123
│   ├── ppocrv6_mobile_rec.onnx   5.4 MB  sha256:def456
│   ├── ppocrv6_cls.onnx         1.8 MB  sha256:ghi789
│   └── model_manifest.json            版本号 + 校验和
│
├── updated/                    # 在线更新的新版本（用户态可写）
│   ├── ppocrv6_mobile_det.onnx
│   ├── ppocrv6_mobile_rec.onnx
│   └── update_manifest.json
│
└── staging/                    # 下载临时目录（原子替换）
    └── *.onnx.download
```

**加载优先级** (registry.rs):

```rust
pub struct ModelRegistry {
    bundled_dir: PathBuf,   // ~/.chana/models/ocr/bundled/
    updated_dir: PathBuf,   // ~/.chana/models/ocr/updated/
    remote_url: Url,        // GitHub Release artifacts / ModelScope
}

impl ModelRegistry {
    /// 获取有效的模型路径（更新版 > 内嵌版）
    pub fn resolve(&self, model_id: &ModelId) -> Result<PathBuf> {
        // 1. 优先使用 updated 版本（用户在线更新过）
        if let Some(path) = self.find_in_updated(model_id) {
            return Ok(path);
        }
        // 2. 回退到 bundled 版本（安装包内嵌）
        if let Some(path) = self.find_in_bundled(model_id) {
            return Ok(path);
        }
        // 3. 都不存在 = 安装包损坏，尝试紧急下载
        Err(ModelError::NotFound)
    }

    /// 自动更新检查（后台 async，不阻塞启动）
    pub async fn check_updates(&self) -> Option<UpdateTask> {
        let remote = self.fetch_remote_manifest().await.ok()?;
        let local = self.local_manifest()?;

        if remote.version > local.version {
            Some(UpdateTask {
                models: remote.diff(&local),    // 仅下载变更的 .onnx
                total_size: remote.total_size(),
            })
        } else {
            None  // 已是最新，无需更新
        }
    }
}
```

**更新流程** (downloader.rs):

```rust
pub struct ModelDownloader {
    client: reqwest::Client,
    registry: ModelRegistry,
}

impl ModelDownloader {
    /// 后台增量更新，不阻塞用户使用
    pub async fn update_async(&self, task: UpdateTask) -> Result<()> {
        // 1. 逐文件下载到 staging/
        for model in &task.models {
            let temp_path = self.registry.staging_dir.join(
                format!("{}.download", model.filename)
            );
            self.download_with_resume(&model.url, &temp_path).await?;

            // 2. 校验 SHA256
            self.verify_checksum(&temp_path, &model.sha256)?;
        }

        // 3. 原子替换：staging/ → updated/（rename 在单文件系统内是原子的）
        for model in &task.models {
            let src = self.registry.staging_dir.join(...);
            let dst = self.registry.updated_dir.join(model.filename);
            std::fs::rename(&src, &dst)?;
        }

        // 4. 写入新 manifest，标记「下次启动生效」
        self.registry.write_updated_manifest(&task.version)?;

        // 5. 系统通知用户
        notify_user("PaddleOCR 模型已更新至 v{version}，重启后生效");

        Ok(())
    }
}
```

**首次启动安装器** (`installer.rs`):

```rust
/// 从应用 bundle 复制模型到用户目录，解耦应用和用户数据
pub fn install_bundled_models() -> Result<()> {
    let bundled = app_resource_dir().join("models");
    let target = user_data_dir().join("models/ocr/bundled");

    if !target.exists() {
        // 首次运行：复制内嵌模型到 ~/.chana/models/ocr/bundled/
        copy_dir_all(&bundled, &target)?;
        info!("PaddleOCR 模型已安装到 {}", target.display());
    }
    // 已存在：跳过，不做覆盖（用户可能有 updated 版本）
    Ok(())
}
```

**关键设计原则**:
- **bundled/** 只读：安装包内嵌，永不修改，作为离线兜底
- **updated/** 可写：在线更新写入此处，优先级高于 bundled
- **原子替换**：通过 `rename` 在 staging→updated 之间切换，避免文件损坏
- **断点续传**：大文件下载支持 Range 请求续传
- **静默更新**：后台下载，不阻塞用户截图，完成时通知

## 三、关键技术选型理由

### 3.1 为什么 Rust？
| 理由 | 说明 |
|------|------|
| **零成本抽象** | FFI调用OS API无额外开销 |
| **内存安全** | 无GC、无Segfault，截图工具不能崩溃 |
| **跨平台** | 条件编译(target_os)天然支持 |
| **包体积** | 静态链接，单文件分发(~15MB) |
| **生态** | xcap/screencapturekit-rs 等已成熟 |

### 3.2 为什么 egui？
| 理由 | 说明 |
|------|------|
| **即时模式** | 每帧重绘，天然适合截图覆盖层 |
| **轻量** | 无复杂状态管理，代码量少 |
| **原生渲染** | glowy(wgpu)后端，非WebView |
| **Rust原生** | 无需JS/Bridge，纯Rust |

### 3.3 为什么 xcap？
- 已封装 Win(DXGI+WGC) / Mac(SCKit+CG) / Linux(PipeWire+X11) 三端差异
- 支持 Monitor/Window 枚举
- 支持 Video Recording（仍在完善中但方向对）
- Apache 2.0 协议友好
